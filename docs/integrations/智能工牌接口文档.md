# 一、开放接口（主动查询）

接口路径: `https://api.yuehangyi.com`

## 请求授权

**POST** `/auth/token`

签名的获取方式：`AccessKey+AccessSecret+Version+Timestamp+Action` 使用 MD5 加密生成签名

| 参数名       | 类型    | 必填 | 说明                       |
| ------------ | ------- | ---- | -------------------------- |
| accessKey    | string  | 是   | 客户端标识 (AppId)         |
| accessSecret | string  | 是   | 密钥 (AppSecret)           |
| version      | string  | 是   | 版本号，默认 v1            |
| timestamp    | integer | 是   | 时间戳，单位秒             |
| action       | string  | 是   | 执行类型，默认 CreateToken |
| signature    | string  | 是   | MD5 签名                   |

# 二、数据推送（平台主动推送到我们）

## 通用推送格式

所有推送为 `POST` JSON，统一结构：

```json
{
  "dataType": "HeartbeatLog",
  "timeStamp": "2025-10-10T01:37:00Z",
  "deviceNo": "BG86384208961234",
  "data": { ... }
}
```

| 字段名    | 类型   | 说明                         |
| --------- | ------ | ---------------------------- |
| dataType  | string | 数据类型                     |
| timeStamp | string | 时间戳                       |
| deviceNo  | string | 设备编号                     |
| data      | object | 业务数据（按 dataType 不同） |

## 鉴权模式

- **V1 明文**: 无额外请求头，明文 JSON
- **V2 带签名**: 请求头 `Api-Version, App-Id, Request-Id, Timestamp, Sign`，Sign = MD5(timestamp+requestId+appId+appSecret)
- **V3 签名+加密**: form-data 传输，录音文件放 `soundRecording`，JSON 数据 AES-CBC-PKCS7 加密后放 `data`

## 推送类型

### 1. Audio — 录音文件推送

录音文件通过 `multipart/form-data` 传输，`soundRecording` 字段放二进制文件。

  "timeStamp": "...",

### 9. MergeAudio — 录音合并

平台自动合并多段切片后的通知。合并机制：

> 同一设备连续录音时，每段切片（A001, A002, ...）独立上传。当**上传间隔超过 15 分钟**时，平台将已上传的同一批次切片合并为一个完整音频文件，推送 MergeAudio。后续恢复录音则产生新批次，再次合并推送。
>
> **命名规则**: `{设备号}-{开始时间}{结束时间}-{分段标识}{序号}.mp3`
> - 切片: `A`=自动分段(录制中) `Z`=结束分段(按停止/关机)
> - 合并: 文件名由最后一片段决定（如带 `Z001`），`fileDownLoadUrl` 路径含 `/merge-audio/`

```json
{
  "dataType": "MergeAudio",
  "deviceNo": "BG863842089611234",
  "data": {
    "deviceNo": "BG863842089611234",
    "fileDownLoadUrl": "https://xxxxx.com/merge/...mp3",
    "fileName": "BG863842089611234-250929...mp3",
    "size": 576441,
    "duration": 217080,
    "extension": ".mp3",
    "startTime": "2025-09-29T10:27:38",
    "endTime": "2025-09-29T10:31:15",
    "objectStoreUrl": "/audio/BG863842089611234/...mp3"
  }
}
```
