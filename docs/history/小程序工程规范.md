## 二、关键规范

### 2.1 样式

- **UnoCSS** 是主力样式方案，使用原子化类名
- 图标：使用 `@iconify-json/ph`（Carbon 图标集），通过 `<view class="i-ph-user-bold" />` 引入
- 本地 SVG 图标放在 `src/static/my-icons/`，UnoCSS 自动扫描
- 禁止直接写死颜色，使用 UnoCSS 主题变量（如 `text-primary` → `var(--wot-color-theme)`）
- 安全区域：`p-safe`、`pt-safe`、`pb-safe` → `env(safe-area-inset-*)`

### 2.2 函数

- **优先使用箭头函数**，禁止使用 `function` 声明式函数
- 页面和组件使用 `<script setup lang="ts">` Composition API 写法

### 2.3 组件（uView Pro）

- 组件通过 `easycom` 自动解析：`^u-(.*)` → `uview-pro/components/u-$1/u-$1.vue`
- 使用 uView Pro 组件前**必须**先查阅 `.agents/skills/uview-pro/SKILL.md`
- 部分组件名称与文档不符（如文档说 `u-cell` 实际是 `u-cell-item`）
- 避免使用 skills 中不存在的组件（如 `datetime-picker` 不存在，应使用原生 picker）

### 2.4 时间处理

使用 `dayjs`，禁止直接拼接字符串。构造时间时用 dayjs 对象操作，不用模板拼接。

```typescript
// 正确：对象操作
dayjs().year(2026).month(6).date(8).hour(10).minute(0).startOf('minute')

// 正确：格式化输出
dayjs().format('YYYY-MM-DD HH:mm')

// 错误：字符串模板拼接
`${year}-${month}-${day} ${hour}:${minute}`
dayjs(`${year}-${month}-${day} ${hour}:${minute}`)
```

### 2.5 请求

统一使用 `src/http/http.ts`（基于 `uni.request` 的轻量封装），JWT 自动刷新和并发请求队列由其内部处理。其他请求封装（如 alova）不再使用，可删除。

### 2.5 请求

- 主力是 `src/http/http.ts`（基于 `uni.request` 的轻量封装）
- JWT 自动刷新：401 时自动刷新 token 并重试，并发请求排队等待
- 统一响应格式：`{ code: 0|200, msg, data }`

### 2.6 路由

- 页面放在 `src/pages/` 下，文件名即路由路径
- `definePage({ type: 'home' })` 标记首页
- 分包目录通过 `subPackages` 配置（如 `src/pages-demo`）
- 自定义 tabbar 通过 `src/tabbar/config.ts` 统一配置

## 三、常用命令

```bash
pnpm dev:mp-weixin      # 微信小程序开发
pnpm build:mp           # 微信小程序构建
pnpm dev:h5             # H5 开发
pnpm dev:app            # App 开发
pnpm type-check         # TypeScript 类型检查
pnpm lint               # ESLint 检查
pnpm lint:fix           # ESLint 自动修复
pnpm test:run           # 运行单元测试
pnpm openapi            # 从后端 Swagger 生成 API 类型
```
