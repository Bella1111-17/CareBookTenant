# Issue 追踪器

**类型：** GitHub Issues

本仓库的 Issues 托管于 [GitHub Issues](https://github.com/saodong7758/CareBook/issues)。

**PR 作为需求入口：** 是

外部 Pull Request 视为需求入口之一。`/triage` 会将外部 PR 与 Issue 一同处理。协作者的在途 PR 不受影响。

---

## 工作流

1. 功能请求或 Bug 报告 → GitHub Issue
2. 实现通过 PR → PR 反向链接 Issue
3. `/triage` 通过同一套标签/状态工作流处理 Issue 和外部 PR
