# CareBook 项目运维辅助信息

> 本文只记录部署拓扑、路径和检查步骤，不保存 SSH、面板、数据库、OSS、第三方平台等明文账号密码。真实凭据请放在服务器环境变量或未纳入 Git 的 `.env.production` 中，并通过受控密码库保存。

## 当前站点

| 项目 | 值 |
| --- | --- |
| 管理后台域名 | `admintenant.care.zbcare.cn` |
| 前端站点目录 | `/www/wwwroot/admintenant.care.zbcare.cn` |
| 后端服务目录 | `/www/wwwroot/admintenant.care.zbcare.cn/server` |
| 后端监听端口 | `8081` |

| 公开访问地址 | `https://admintenant.care.zbcare.cn` |
| Swagger 地址 | `https://admintenant.care.zbcare.cn/prod-api/swagger-ui/` |

## 数据库

| 项目 | 值 |
| --- | --- |
| 类型 | PostgreSQL |
| 地址 | 通过 `TOB_POSTGRES_HOST` / `TOB_POSTGRES_PORT` 配置 |
| 用户 | 通过 `TOB_POSTGRES_USER` 配置 |
| 密码 | 通过 `TOB_POSTGRES_PASSWORD` 配置 |
| 数据库 | 通过 `TOB_POSTGRES_DATABASE` 配置 |

生产环境至少需要配置：

```bash
TOB_POSTGRES_HOST=101.133.170.55
TOB_POSTGRES_PORT=5432
TOB_POSTGRES_USER=tenant_admin
TOB_POSTGRES_PASSWORD=<在服务器填写>
TOB_POSTGRES_DATABASE=carebook_tenant
TOB_JWT_SECRET=<在服务器填写>
```

完整示例见仓库根目录 `.env.production.example`。

## 部署步骤

1. 在 DNS 服务商确认 `admintenant.care.zbcare.cn` 解析到服务器公网 IP。
2. 在宝塔/Nginx 创建站点，站点目录使用 `/www/wwwroot/admintenant.care.zbcare.cn`。
3. 在服务器创建后端目录 `/www/wwwroot/admintenant.care.zbcare.cn/server`。
4. 在服务器后端目录放置生产环境文件 `.env`，或本地准备未跟踪的 `.env.production` 后执行部署脚本上传。
5. 初始化或迁移远程 PostgreSQL，执行前先备份，SQL 位于 `apps/server/db/` 和 `scripts/`。
6. 部署后端：`bash deployment/server-backend.sh`。
7. 部署前端：`bash deployment/admin-frontend.sh`。
8. 检查 `https://admintenant.care.zbcare.cn`、`/prod-api/swagger-ui/` 和登录接口。

如果需要指定生产环境文件：

```bash
DEPLOY_ENV_FILE=/secure/path/carebook-tenant.env bash deployment/server-backend.sh
```

## Nginx 反代建议

前端静态资源由站点根目录提供，后端接口走同域 `/prod-api/`：

```nginx
location /prod-api/ {
    proxy_pass http://127.0.0.1:8081/prod-api/;
    proxy_set_header Host $host;
    proxy_set_header X-Real-IP $remote_addr;
    proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
    proxy_set_header X-Forwarded-Proto $scheme;
}

location /upload/ {
    proxy_pass http://127.0.0.1:8081/upload/;
    proxy_set_header Host $host;
}
```

## 智能工牌回调

悦航益平台生产回调地址应配置为：

```text
https://admintenant.care.zbcare.cn/prod-api/hardware/badge/callback
```

相关环境变量：

```bash
TOB_HARDWARE_APP_ID=<在服务器填写>
TOB_HARDWARE_APP_SECRET=<在服务器填写>
TOB_HARDWARE_CALLBACK_SECRET=<在服务器填写，如平台无签名可留空>
TOB_OSS_DOMAIN=https://carebook.cn-beijing.taihangpkx.cn
```

## 安全事项

- 不要把 `.env`、`.env.production`、服务器登录密码、面板密码、OSS AccessKey、第三方平台账号密码提交到 Git。
- 如果凭据曾出现在聊天、文档或提交历史中，应按生产事故处理并立即轮换。
- 数据库脚本执行前先备份，涉及清理脚本时记录回滚方案。
- 本地开发连接远程生产库有误操作风险，执行写入类脚本前必须二次确认目标库。
