# 服务端规范（apps/server）

NestJS API 服务，端口 8080，PostgreSQL 16 + Redis。

## 二、核心规范

### 2.1 模块注册

NestJS CLI 的 module 扫描已禁用（`nest-cli.json` 中 `generateOptions.spec: false, flat: false`）。所有模块必须手动 import 到 `app.module.ts`。

### 2.2 装饰器速查

位于 `src/common/decorators/`：

| 装饰器 | 文件 | 用途 |
|--------|------|------|
| `@notRequireAuth` | `common.decorator.ts` | 跳过 JWT 鉴权（加在 Controller 方法上） |
| `@RequirePermission('xxx')` | `require-premission.decorator.ts` | 权限校验，需配合 `PermissionGuard` |
| `@RequireRole('xxx')` | `require-role.decorator.ts` | 角色校验，需配合 `RolesGuard` |
| `@DataScope` | `common.decorator.ts` | 数据权限过滤（五级：全部/本部门/本部门及子部门/自定义/仅本人） |
| `@Log({ businessType: BusinessType.XXX })` | `operlog.decorator.ts` | 操作日志记录（自动记录操作人、时间、IP、业务类型） |
| `@Task({ name: 'task.xxx', description: '...' })` | `task.decorator.ts` | 定时任务注册（注册到 `TaskRegistry`，由 `TaskService` 统一调度） |
| `@Cacheable('缓存名', 'key', 过期秒数)` | `redis.decorator.ts` | 方法级 Redis 缓存 |
| `@CacheEvict('缓存名', 'key')` | `redis.decorator.ts` | 缓存淘汰（支持 `*` 清空前缀） |
| `@Captcha('key')` | `captcha.decorator.ts` | 验证码校验（依赖 Redis + SysConfigService） |
| `@ApiDataResponse(Model, isArray?, isPager?)` | `apiDataResponse.decorator.ts` | Swagger 响应包装（自动生成 `{code,msg,data}` 结构） |
| `@ClientInfo` | `common.decorator.ts` | 取当前请求的客户端信息（IP、浏览器、OS、用户名） |

### 2.3 实体基类

位于 `src/common/entities/base.ts`，三层抽象：

```
DeleteStatusEntity        → delFlag ('0'=正常,'1'=已删)
       ↓
BaseStatusEntity          → + status ('0'=正常,'1'=停用)
       ↓
BaseEntity                → + createBy / createTime / updateBy / updateTime / remark
```

所有业务实体继承 `BaseEntity`。软删除通过 `delFlag` 实现，不用 `@DeleteDateColumn`。

### 2.4 鉴权流程

三层守卫，全局生效（通过 `APP_GUARD`）：

1. `JwtAuthGuard` — 验证 JWT 有效性，白名单路由 bypass，`@notRequireAuth` 跳过
2. `RolesGuard` — 检查 `@RequireRole` 装饰器
3. `PermissionGuard` — 检查 `@RequirePermission` 装饰器，通配 `*:*:*` 全通过

### 2.5 响应格式

所有接口统一返回 HTTP 200，响应体：

```json
{ "code": 200, "msg": "操作成功", "data": {} }
```

- `code` 字段映射 HTTP 状态码
- 异常由 `http-exceptions-filter.ts` 统一捕获，返回结构化错误响应
- 前端 axios 拦截器按 `code` 值分支（401 自动刷新 token、500 报错、601 特殊错误）

### 2.6 数据库约定

- 仅 PostgreSQL 16，不用 MySQL
- 金额字段：整数分存储（如 `30000` = 300 元）
- 布尔标志位：`char(1)` 存 `'0'`/`'1'`，不用 `BOOLEAN`
- 时间字段：`timestamp`，不用 `datetime`
- 所有实体使用 `@PrimaryGeneratedColumn({ type: 'int' })`
- Service 注入 Repository：`@InjectRepository(Entity)`，不用 `EntityManager`
- 复杂查询用 `createQueryBuilder`，简单 CRUD 用 `repository.save()/update()/delete()`
- 多对多关系用显式中间表实体，不用 `@ManyToMany`
- 软删除过滤优先用 `createQueryBuilder`，显式加 `.andWhere('alias.delFlag = :delFlag', { delFlag: '0' })`
- **状态判断和软删除判断必须使用枚举**，禁止硬编码字符串 `'0'`/`'1'`

### 2.7 DTO 规范

每个业务模块 `dto/index.ts` 统一导出以下 DTO：

```typescript
// 创建 DTO
export class CreateXxxDto {
  name: string;
  // ...
}

// 更新 DTO，继承 Create 并设为可选
export class UpdateXxxDto extends PartialType(CreateXxxDto) {}

// 列表查询 DTO，继承 PagingDto
export class ListXxxDto extends PagingDto {
  name?: string;
  // ...
}
```

### 2.8 配置加载

`src/config/index.ts` 读取 `{dev,test,prod}.yml`，由 `@nestjs/config` 全局注入。`nest-cli.json` 将 `**/*.yml` 拷贝到构建输出目录。

### 2.9 工具函数

位于 `src/common/utils/`，统一使用这些函数，禁止自行重复实现：

| 函数 | 用途 |
|------|------|
| `GetNowDate()` | 返回 `YYYY-MM-DD HH:mm:ss` 格式当前时间 |
| `FormatDate(date, format?)` | 时间格式化，支持自定义格式，默认 `YYYY-MM-DD HH:mm:ss` |
| `dateTransformer` | TypeORM 值转换器，数据库 timestamp → `YYYY-MM-DD HH:mm:ss` |
| `ListToTree(arr, getId, getLabel)` | 数组转树形结构 |
| `DeepClone(obj)` | 深拷贝（基于 lodash） |
| `GenerateUUID()` | 生成无横线 UUID |
| `Paginate(data, pageSize, pageNum, filterParam)` | 内存分页（用于日志等小数据量场景） |
| `isEmpty(value)` | 判断 null / undefined / 空字符串 / NaN |
| `ResultData.ok(data, msg)` | 返回成功响应 `{ code: 200, msg, data }` |
| `ResultData.fail(code, msg, data)` | 返回失败响应 `{ code, msg, data }` |
| `ExportTable(options, res)` | 通用 Excel 导出，支持字典映射和自定义格式化 |

## 三、新建流程

### 3.1 新建 Module

1. 在 `src/module/{system|monitor|wechat}/<name>/` 下创建 `*.module.ts`、`*.service.ts`、`*.controller.ts`
2. 在 `dto/` 下创建 `index.ts` 导出所有 DTO
3. 在 `entities/` 下创建 `*.entity.ts`
4. 在父模块中 import
5. 在 `app.module.ts` 中 import 父模块

### 3.2 新建定时任务

`@Task` 装饰器**统一放在 `task.service.ts`**，业务 Service 只写纯业务逻辑，通过 `ModuleRef` 调用。

1. 在业务 Service 中编写业务逻辑方法（如 `XxxService.doSomething()`）
2. 在 `task.service.ts` 中用 `@Task` 包装，委托调用业务 Service：

```typescript
@Task({ name: 'task.xxx', description: '任务描述' })
async xxxTask() {
  // 通过 moduleRef 动态获取业务 Service
  const xxxService = await this.moduleRef.get(XxxService, { strict: false });
  await xxxService.doSomething();
}
```

3. 在 `sys_job` 表中插入任务记录

```typescript
@Task({ name: 'task.concatDailyTranscript', description: '合并每日转写' })
async concatDailyTranscript() {
  await this.hardwareAudioService.concatDailyTranscript();
}
```

3. 在 `sys_job` 表中插入任务记录：

```sql
INSERT INTO sys_job (job_name, job_group, invoke_target, cron_expression, status, create_by)
VALUES ('合并每日转写', 'DEFAULT', 'task.concatDailyTranscript', '0 0 7 * * ?', '1', 'admin');
```

### 3.3 Admin 新建页面（菜单表）

Admin 页面路由依赖 `sys_menu` 表记录，动态路由从后端 `/getRouters` 获取。

**步骤：**

1. 在 `apps/admin-vue3/src/views/` 下创建 Vue 页面文件
2. 在 `sys_menu` 表中插入菜单数据：

```sql
INSERT INTO sys_menu (
  menu_name, parent_id, order_num, path, component, query,
  menu_type, visible, is_frame, is_cache, perms, icon,
  create_by, update_by, del_flag, status
) VALUES (
  '用户管理', 0, 1,
  'system/user',          -- 路由地址
  'system/user/index',    -- 组件路径
  '',                     -- 路由参数
  'C',                    -- M=目录 C=菜单 F=按钮
  '0',                    -- 0=显示 1=隐藏
  '1',                    -- 0=外链 1=内部
  '0',                    -- 0=缓存 1=不缓存
  'system:user:list',     -- 权限标识（按钮级菜单填）
  'user',                 -- 图标
  'admin', '', '0', '0'
);
```

**component 路径规则：**
- 视图文件：`system/user/index` → `src/views/system/user/index.vue`
- 外链：填写完整 URL 如 `http://xxx.com`

**perms 权限标识规则（按钮级）：**
- 列表 `system:user:list` / 查询 `system:user:query` / 新增 `system:user:add`
- 修改 `system:user:edit` / 删除 `system:user:remove` / 导出 `system:user:export`
