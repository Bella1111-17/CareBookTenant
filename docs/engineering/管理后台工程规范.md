## 二、关键规范

### 2.1 请求与认证

- API 请求通过 `/dev-api` 代理到 `http://localhost:8080`
- JWT 放在 `Authorization: Bearer <token>` Header
- 动态路由从后端 `/getRouters` 获取，组件路径通过 `import.meta.glob` 解析
- Token 刷新：401 时 axios 拦截器自动刷新，重试失败则跳转登录页

### 2.2 图片/文件字段处理

- `ImageUpload` 和 `FileUpload` 组件通过 `v-model` 发出**逗号分隔字符串**，不是数组
- 多图字段在表单/data 层存原始字符串，按需在前端 UI 层转为数组展示
- 单图字段（`:limit="1"`）存储的仍然是字符串，不是数组

### 2.3 权限指令

```html
<!-- 权限标识符校验 -->
<el-button v-hasPermi="['system:user:add']">新增</el-button>

<!-- 角色校验 -->
<el-button v-hasRole="['admin']">管理员专用</el-button>
```

### 2.4 图标使用

- Element Plus 内置图标：`@element-plus/icons-vue`
- 用法：`<el-icon><User /></el-icon>`
- 或在菜单/按钮的 `icon` 字段填图标名称（如 `user`）

### 2.5 新建页面流程

1. 在 `src/views/` 下创建 Vue 页面文件
2. 在数据库 `sys_menu` 表中插入菜单数据（参见 `docs/engineering/服务端规范.md` 3.3 节）
3. 刷新页面，路由自动加载

### 2.6 Store 规范（Pinia）

统一使用 Pinia Setup 语法（Composition API 风格），禁止使用 Options API：

```typescript
// 正确
import { defineStore } from 'pinia'
export const useUserStore = defineStore('user', () => {
  const token = ref('')
  const roles = ref<string[]>([])

  const setToken = (t: string) => { token.value = t }
  const login = async (params) => { /* ... */ }

  return { token, roles, setToken, login }
})

// 错误：Options API 风格
export const useUserStore = defineStore('user', {
  state: () => ({ token: '', roles: [] }),
  actions: { setToken(t) { this.token = t } }
})
```
